## 1. shadcn/ui + Tailwind CSS v4 Compatibility

### Status

shadcn/ui is fully compatible with Tailwind CSS v4 via its latest CLI (`npx shadcn@latest`).

### Key Differences from v3

| Feature | Tailwind v3 | Tailwind v4 |
|---------|------------|------------|
| Config File | `tailwind.config.js` required | Removed. Config in CSS via `@theme {}` |
| Directives | `@tailwind base/components/utilities` | `@import "tailwindcss";` |
| Vite Plugin | PostCSS config | `@tailwindcss/vite` plugin |
| Theme Extension | JS `extend: { ... }` | CSS `@theme { ... }` |
| Animations | `tailwindcss-animate` plugin | Native CSS or `tw-animate-css` |

### Setup Steps

1. **Path Alias**: Add `@` alias to both `tsconfig.json` and `vite.config.ts`:

   ```typescript
   // vite.config.ts
   import path from "path"

   export default defineConfig({
     // ...
     resolve: {
       alias: {
         "@": path.resolve(__dirname, "./src"),
       },
     },
   })
   ```

   ```json
   // tsconfig.json - add to compilerOptions
   {
     "baseUrl": ".",
     "paths": {
       "@/*": ["./src/*"]
     }
   }
   ```

2. **Initialize shadcn/ui**:

   ```bash
   npx shadcn@latest init
   ```

   Options:
   - Style: New York (recommended)
   - Base Color: Slate (or preference)
   - CSS Variables: Yes (critical for v4)
   - Global CSS: `src/index.css`

3. **CSS Configuration** (auto-generated by init):

   ```css
   @import "tailwindcss";

   @plugin "tailwindcss-animate";
   @custom-variant dark (&:is(.dark *));

   @theme {
     --radius: 0.5rem;
     --color-background: var(--background);
     --color-foreground: var(--foreground);
     --color-primary: var(--primary);
     --color-primary-foreground: var(--primary-foreground);
     /* ... other semantic color mappings ... */
   }

   :root {
     --background: 0 0% 100%;
     --foreground: 222.2 84% 4.9%;
     --primary: 222.2 47.4% 11.2%;
     --primary-foreground: 210 40% 98%;
     /* ... */
   }

   .dark {
     --background: 222.2 84% 4.9%;
     --foreground: 210 40% 98%;
     /* ... */
   }
   ```

4. **Add components**:

   ```bash
   npx shadcn@latest add button card table badge
   ```

## 2. Vite 6 Proxy Configuration for FastAPI

### Configuration

```typescript
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";
import tailwindcss from "@tailwindcss/vite";
import path from "path";

export default defineConfig({
  plugins: [react(), tailwindcss()],
  resolve: {
    alias: {
      "@": path.resolve(__dirname, "./src"),
    },
  },
  server: {
    proxy: {
      "/api": {
        target: "http://localhost:3000",
        changeOrigin: true,
        secure: false,
        ws: true,
        // Optional: Rewrite path if backend doesn't use /api prefix
        // rewrite: (path) => path.replace(/^\/api/, ''),
      },
    },
  },
  build: {
    outDir: "../src/insight_blueprint/static",
    emptyOutDir: true,
  },
});
```

### Key Points

- **`changeOrigin: true`**: Changes Host header to match target. Required for some backend checks.
- **`ws: true`**: Enable WebSocket proxying if needed.
- **CORS**: Proxy eliminates need for CORS config in development (browser sees same origin).
- **Path Rewriting**: Only needed if FastAPI routes don't include `/api` prefix. Since our backend uses `APIRouter(prefix="/api")`, do NOT rewrite.
- **`secure: false`**: Safe for local HTTP development.
